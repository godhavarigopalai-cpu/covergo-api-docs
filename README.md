# CoverGo API Documentation

This is the MkDocs documentation project for the CoverGo GraphQL API.

## Overview

The CoverGo API provides GraphQL-based access to insurance and financial services. This documentation was generated by analyzing the GraphQL schema at `https://api.dev.covergo.cloud/graphql/`.

## Important Findings

!!! success "Authentication APIs Found"
    The schema analysis discovered several authentication and token management APIs:

    **Token Management Queries:**
    - `token_2` - Token refresh functionality
    - `refreshToken` - Token refresh operations
    - `verifyGOODTokenAndGetCoverGoToken` - Token verification
    - SSO token queries for enterprise authentication

    **Authentication Mutations:**
    - `createAccessTokenFromOtpLogin` - OTP-based authentication
    - `generateKycAuthToken` - KYC process authentication
    - `initializeTenantAuth` - Tenant authentication setup

    The schema also contains 50+ business operations for insurance and financial services.


2. Serve documentation locally:

3. Build documentation:

## Schema Analysis Methodology

The documentation was created by:

1. **GraphQL Introspection**: Querying the schema at `https://api.dev.covergo.cloud/graphql/`
2. **Mutation Analysis**: Extracting all available mutation operations
3. **Documentation Generation**: Creating comprehensive guides and references

## Available APIs

The schema contains 50+ mutation operations for:

- **Billing**: Channel management, payor operations
- **Finance**: Fee management, IBAN mappings, GL operations
- **Policies**: Premium management, cancellations
- **Payments**: Invoices, receipts, SEPA operations
- **Reference Data**: CRUD operations for reference data

See [Available Mutations](docs/api-reference/mutations.md) for the complete list.
